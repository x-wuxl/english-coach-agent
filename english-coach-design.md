# 2026-03-24 英语学习教练设计方案

## 1. 项目目标

构建一个面向泛英语学习用户的 Agent 英语教练。它不是单纯背词工具，而是一个会根据用户水平、记忆状态、学习目标与实时表现，持续生成和优化个性化学习路径的学习系统。

第一阶段以聊天式教练为核心体验，后续可扩展为带页面的轻应用。

核心目标：
- 判断用户当前英语水平与学习目标
- 自动生成每日学习与复习任务
- 结合艾宾浩斯规律与真实学习表现动态调整复习节奏
- 通过错因分析、反馈与节奏控制提升长期坚持率
- 支持提醒、复盘、学习记录与后续工具接入

---

## 2. 产品定位

### 2.1 产品形态
第一版产品形态为“教练体验 + 规则引擎混合型”。

用户看到的是一个对话式英语教练；底层运行的是一组可解释、可扩展的学习引擎模块，包括定级、计划、记忆调度、反馈分析与复盘。

### 2.2 目标用户
第一版定位为泛英语学习用户，不限定考试、口语或职场中的单一赛道，但底层保留按目标切换学习策略的能力。

### 2.3 产品原则
1. 任务流优先，不做纯内容堆砌
2. 复习优先级由遗忘风险和真实表现决定
3. 输出能力必须进入学习主流程
4. 核心逻辑先跑通，工具能力采用适配层预留接口
5. AI 用于增强学习体验，不接管核心调度

---

## 3. 核心工作流

### 3.1 首次进入
1. 收集用户目标、可投入时间、学习习惯
2. 进行轻量定级测试
3. 生成能力画像
4. 生成当天或次日第一份学习计划
5. 进入第一轮学习任务

### 3.2 每日学习
1. 开场说明今日学习任务
2. 热身复习高风险旧内容
3. 新学当前主题内容
4. 即时检测
5. 输出强化（造句/中译英/情景补全）
6. 收尾总结并更新后续节奏

### 3.3 轻复习模式
适用于碎片时间或状态较差时：
- 快速复习高风险内容
- 处理昨日错题
- 完成少量输出练习

### 3.4 每周复盘
输出内容包括：
- 本周完成率
- 学习总时长
- 新学/复习数量
- 高频错误类型
- 最稳主题与最弱主题
- 下周节奏与内容建议

---

## 4. 首次定级设计

### 4.1 定级目标
定级不只输出综合等级，还要构建多维能力画像：
- 综合水平
- 词汇识别
- 语法稳定度
- 阅读理解
- 输出能力
- 当前主要短板
- 初始学习节奏建议

### 4.2 定级输入
- 学习目标（泛学习/考试/口语/阅读/职场等）
- 每日可学习时长
- 自我感知水平
- 学习习惯与时间偏好

### 4.3 定级方式
建议控制在 5~8 分钟，采用混合评估：
- 词汇快测：词义、短语、语境猜词
- 语法快测：时态、介词、冠词、语序、主谓一致等
- 阅读理解：短文 + 理解题
- 输出小对话：英文简答

### 4.4 定级输出示例
- 综合水平：A2+
- 词汇识别：B1
- 语法稳定度：A2
- 阅读：A2+
- 输出：A1+
- 主要问题：会认不会用、时态不稳、搭配薄弱

## 5. 系统模块设计

### 5.1 定级器 Placement Engine
职责：
- 首次定级
- 阶段性重估
- 输出用户能力画像与初始节奏建议

### 5.2 学习计划器 Daily Planner
职责：
- 根据用户画像、记忆状态、最近表现与可用时长生成当日任务包
- 决定学什么、学多少、按什么顺序学

### 5.3 记忆调度器 Memory Scheduler
职责：
- 计算每个学习项的遗忘风险
- 排定复习优先级
- 决定下次复习时间
- 推荐下次练习方式

### 5.4 学习执行器 Session Coach
职责：
- 以教练式对话带用户完成任务流
- 控制学习节奏
- 过程内即时反馈

### 5.5 反馈分析器 Feedback Analyzer
职责：
- 不只记录对错，还分析错因类型
- 识别用户的薄弱模式
- 为计划器与调度器提供动态修正依据

### 5.6 周复盘器 Weekly Review
职责：
- 统计一周行为与表现
- 判断是否需要加量、减量、切换训练方式
- 输出下周策略建议

### 5.7 AI 增强层 LLM Assist Layer
职责：
- 将结构化结果转成自然教练式反馈
- 生成例句、微场景、小测验草稿
- 生成错因解释和学习总结
- 后续支持自由对话陪练与开放式表达点评

边界：
- 不负责核心定级评分
- 不直接决定复习调度
- 不直接更新掌握状态主字段

### 5.8 工具适配层 Tool Adapter
职责：
- 统一封装提醒、词典、发音、测验、统计等能力
- 第一版先定义接口，后续逐步接入真实工具

## 6. 数据模型设计

### 6.1 UserProfile
记录用户画像、目标与偏好。

建议字段：
- user_id
- goal
- sub_goals
- daily_minutes
- study_start_time
- review_time
- overall_level
- vocab_level
- grammar_level
- reading_level
- output_level
- preferred_modes
- motivation_style
- fatigue_tolerance

### 6.2 LearningItem
统一抽象单词、短语、短句、语法点、场景表达。

建议字段：
- item_id
- type
- content
- meaning_zh
- difficulty
- theme
- tags
- examples
- related_items

### 6.3 MasteryState
记录用户对某个学习项的真实掌握状态。

建议字段：
- user_id
- item_id
- seen_count
- correct_count
- wrong_count
- correct_streak
- recognition_score
- recall_score
- output_score
- memory_strength
- forget_risk
- last_seen_at
- next_review_at
- last_mode
- recommended_mode
- status

### 6.4 StudySession
记录一次完整学习会话。

建议字段：
- session_id
- user_id
- date
- duration_min
- new_items_count
- review_items_count
- accuracy
- completion_rate
- fatigue_feedback
- mood_feedback
- focus_theme

### 6.5 AttemptLog
记录单题级别的尝试表现。

建议字段：
- attempt_id
- user_id
- item_id
- session_id
- mode
- result
- response_time_ms
- hint_used
- error_type
- timestamp
- llm_explanation_snapshot

### 6.6 学习项状态机
为避免实现时状态混乱，第一版建议为 MasteryState 定义最小状态机：
- new：未学过
- learning：已接触但不稳定
- reviewing：进入规律复习期
- weak_output：识别稳定但输出薄弱
- mastered：稳定掌握
- archived：长期稳定，低频抽查

状态切换原则：
- 新学后默认进入 learning
- 连续识别正确但输出低，进入 weak_output
- 多次稳定正确后进入 reviewing 或 mastered
- 若 mastered 内容连续错误，则回退到 reviewing

### 6.7 内容边界与最小内容集
第一版不追求大而全，建议仅准备可控的最小内容集：
- 200~500 个高频词
- 100~200 个高频短语
- 100 个常用短句/场景表达
- 少量基础语法点
- 明确主题标签（如日常、时间安排、工作沟通、购物、出行）

内容标准：
- 尽量服务表达，而不是只做词义记忆
- 每个学习项至少带一个例句
- 尽量支持同主题聚合与 related_items 关联

## 7. 每日计划生成逻辑

### 7.1 步骤一：确定今日总负载
依据以下因素动态计算：
- 每日可学习时长
- 最近 3 天完成率
- 最近 3 天正确率
- 疲劳反馈
- 连续学习天数

策略：
- 正常状态：新学 6~8，复习 10~15，输出 2~3
- 状态好：小幅加量
- 状态差：减少新学量，优先复习，降低输出难度

### 7.2 步骤二：挑选复习项
优先复习：
1. 已到 next_review_at 且遗忘风险高的内容
2. 最近反复出错内容
3. 识别会但输出不会的内容
4. 当前主题关键表达
5. 少量稳定项抽查

### 7.3 步骤三：挑选新学项
新学项选择依据：
- 与用户目标相关
- 与近期主题连续
- 难度略高于当前稳定区
- 能形成表达组合，而不是孤立词条

### 7.4 步骤四：拼装任务流
建议固定任务顺序：
1. 热身复习
2. 新学输入
3. 即时检测
4. 输出强化
5. 收尾小测
6. 学习总结

---

## 8. 记忆与复习系统设计

### 8.1 基本原则
复习系统采用“基础间隔 + 真实表现修正”模式，而不是死板套用固定曲线。

### 8.2 基础复习间隔
建议起步曲线：
- 学后当天 / 次日
- 第 1 天
- 第 3 天
- 第 7 天
- 第 14 天
- 第 30 天

### 8.3 表现修正规则
如果内容表现优秀：
- 直接答对
- 反应快
- 输出也正确
则延后下次复习。

如果内容表现一般：
- 识别能对
- 输出不稳
则保持或轻微缩短间隔。

如果内容表现较差：
- 答错
- 提示后才会
- 反应明显偏慢
则提前复习，并切换强化方式。

### 8.4 复习优先级计算
第一版可采用可解释的评分：

review_priority =
0.30 * due_score +
0.25 * forget_risk +
0.20 * recent_error_score +
0.15 * output_gap_score +
0.10 * theme_weight -
0.10 * recent_exposure_penalty

说明：
- due_score：是否到复习时间、是否超时
- forget_risk：历史表现与遗忘风险
- recent_error_score：近期错误频率
- output_gap_score：识别高但输出低的差值
- theme_weight：当前主题重要度
- recent_exposure_penalty：刚练过则降权

### 8.5 题型切换逻辑
- 新学：释义、例句、发音、匹配
- 初步掌握：选择题、填空、搭配判断
- 虚会状态：中译英、造句、情景问答
- 反复错误：辨析、错因解释、微场景重练

## 9. AI 角色与边界

### 9.1 第一版 AI 策略
第一版采用“规则引擎为主，LLM 为辅”的策略。

### 9.2 必须规则驱动的部分
以下模块必须由可解释规则主导：
- 定级评分
- 日计划负载计算
- 复习优先级计算
- 掌握状态更新
- 周复盘节奏建议

### 9.3 可以由 AI 增强的部分
以下能力可以由 LLM 增强：
- 教练话术生成
- 错因解释润色
- 例句改写与微场景生成
- 小测题干和对话练习草稿生成
- 学习总结与鼓励反馈
- 后续自由表达点评与对话陪练

### 9.4 AI 调用原则
- LLM 输出不直接写入核心状态字段，除非经过规则校验或结构化约束
- 对同一结构化结果，LLM 主要负责“说法更自然”，而不是“改事实”
- 若 AI 调用失败，主学习流程仍应可继续

### 9.5 AI 适配接口
建议抽象以下适配器：
- CoachLlmAdapter：生成教练反馈与收尾总结
- ExplanationAdapter：根据结构化错因生成解释文本
- ExerciseGenerationAdapter：根据学习项生成例句、微场景和练习草稿

## 10. 反馈与激励机制

### 10.1 反馈层级
每次反馈应包含三层：
1. 结果反馈：对 / 错 / 部分对
2. 错因反馈：词义混淆、语序错误、搭配错误、时态错误、会认不会用等
3. 节奏反馈：是否加量、减量、提前复习、延后复习

### 10.2 鼓励风格
避免机械化“正确/错误”提示，应使用教练式反馈，例如：
- 这个词你已经连续答对 3 次，基本稳了
- 这题是典型搭配问题，不是不会，是用法还没固定
- 今天状态一般，我先帮你减一点新内容，优先把旧的稳住

### 10.3 激励机制
- 连续学习天数
- 阶段性里程碑
- 主题掌握进度
- 错题回收率
- 每周成长反馈

---

## 11. 提醒与节奏管理

### 11.1 支持设置
- 每天开始学习时间
- 复习提醒时间
- 每日目标
- 每周复盘时间

### 11.2 提醒类型
1. 开始学习提醒
2. 轻复习提醒
3. 中断召回提醒
4. 周复盘提醒

### 11.3 节奏自动调节
根据最近表现与疲劳反馈：
- 加量：正确率高、完成率高、主观难度正常
- 减量：错误率高、连续中断、主观难度高
- 换方式：识别会但输出弱时提高输出练习比重

---

## 12. 工具调用与适配层

第一版核心采用本地规则和内容运行，但工具接口需要提前抽象。

建议预留以下适配器：
- ReminderAdapter：学习提醒、复习提醒、周复盘提醒
- DictionaryAdapter：释义、例句、同义词、搭配
- PronunciationAdapter：发音、跟读
- QuizAdapter：题目生成
- StatsAdapter：学习记录与进度统计
- CoachLlmAdapter：教练自然语言输出
- ExplanationAdapter：错因解释输出
- ExerciseGenerationAdapter：练习生成输出

设计原则：
- 核心学习逻辑不直接依赖具体工具实现
- 工具调用失败时不影响主学习流程
- 后续可按需接入真实词典、发音、提醒系统和模型提供方

## 12.5 实现风险与边界说明

### 12.5.1 不把大模型判断当成唯一真相
第一版可以使用规则与模型混合判断，但掌握状态更新应优先依赖结构化结果（对错、反应时、提示、题型），避免完全依赖主观语言评估。

### 12.5.2 输出题不能过早过重
输出是核心，但对低水平用户不能一开始就高压要求自由造句。第一版应允许从填空、替换、半开放式造句逐步过渡。

### 12.5.3 计划生成必须可解释
日计划生成结果应能说明：
- 为什么今天复习这些内容
- 为什么今天新学量是这个数
- 为什么某些内容被提前或延后

这样方便后续调试与用户建立信任。

## 15. 典型用户日流程示例

### 中午
系统提醒用户进行 5 分钟轻复习：
- 优先处理最易遗忘内容
- 快速回收昨日错题

### 晚上正式学习
1. 热身复习旧内容
2. 学习今日新内容
3. 做即时检测
4. 完成输出任务
5. 做收尾小测
6. 查看学习总结

系统记录：
- 哪些内容稳定
- 哪些内容虚会
- 哪些内容需要提前复习

AI 可参与：
- 把错因转成自然解释
- 生成一条更像教练的总结

### 周末
系统生成周复盘：
- 学习时长
- 完成率
- 高频错误
- 节奏调整建议

## 16. 成功标准

第一版成功不看功能数量，而看是否达成以下结果：
- 能稳定完成定级 → 计划 → 学习 → 调整 → 复盘闭环
- 用户能感受到“系统知道我该学什么”
- 系统能识别“会认不会用”和“反复错误”这类真实学习问题
- 系统能根据表现动态调节负载和复习时机
- AI 增强学习体验，但不会让核心调度失控
- 后续可以无痛接入提醒、词典、发音、页面等能力
